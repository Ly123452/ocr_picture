# BOSS直聘信息OCR识别工具

基于计算机视觉和OCR技术的BOSS直聘界面信息自动提取工具，能够智能识别候选人列表、聊天消息和个人信息等关键数据。

## ✨ 核心特性

- **🎯 智能区域检测**：无需硬编码坐标，自适应不同分辨率和界面布局
- **📋 候选人列表识别**：自动检测未读/已读状态，提取姓名和消息预览
- **💬 聊天消息提取**：精准识别对话气泡，区分"我"和"对方"发送的消息
- **👤 个人信息解析**：智能提取姓名、职位、沟通职位、期望等字段
- **🔍 OCR文字识别**：集成RapidOCR高精度引擎（PP-OCRv4模型）
- **📊 双格式输出**：同时生成JSON结构化数据和TXT可读报告

## 🛠️ 技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 图像处理 | OpenCV | 颜色检测、形态学操作、轮廓分析 |
| OCR引擎 | RapidOCR (ONNX Runtime) | 基于PaddleOCR PP-OCRv4高精度模型 |
| 数值计算 | NumPy | 图像数组处理 |
| 图像IO | Pillow | 图片格式支持 |
| Python版本 | 3.8 - 3.11 | 推荐3.10以获得最佳兼容性 |

## 📦 快速开始

### 1. 环境准备

确保已安装Python 3.8-3.11（推荐3.10）：

```bash
python --version
```

### 2. 创建虚拟环境（推荐）

```bash
# Windows
py -3.10 -m venv .venv
.venv\Scripts\activate

# Linux/Mac
python3.10 -m venv .venv
source .venv/bin/activate
```

### 3. 安装依赖

```bash
pip install opencv-python numpy Pillow rapidocr-onnxruntime
```

或使用清华镜像加速：

```bash
pip install opencv-python numpy Pillow rapidocr-onnxruntime -i https://pypi.tuna.tsinghua.edu.cn/simple
```

> ⚠️ **注意**：首次运行时，RapidOCR会自动下载OCR模型文件（约100MB），请保持网络连接。

### 4. 运行程序

```bash
python ocr_recognizer.py
```

按提示输入截图路径，或直接拖拽图片到命令行窗口。

## 🔧 高级配置

### OCR模型选择

默认使用高精度模型（PP-OCRv4）。如需调整，可修改 `_init_ocr()` 方法：

```python
# 快速模式（速度优先）
self.ocr = RapidOCR()

# 高精度模式（精度优先，默认）
self.ocr = RapidOCR(
    det_model_name='ch_PP-OCRv4_det', 
    rec_model_name='ch_PP-OCRv4_rec'
)
```

## ❓ 常见问题

### Q1: 个人信息字段为空？

**原因：**
- OCR未能识别到文字（图片模糊、字体过小）
- 关键词匹配规则不适配当前界面

**解决方案：**
1. 检查 `[DEBUG] 个人信息区域原始OCR结果` 输出
2. 如果OCR结果为空 → 提高截图清晰度
3. 如果OCR有结果但未匹配 → 调整关键词规则（见 `extract_profile_info()` 方法）

### Q2: 聊天消息识别不全？

**原因：**
- 消息气泡检测参数不适合当前布局
- 多行文本被截断

**解决方案：**
1. 调整形态学膨胀核大小（第113行）
2. 增加padding值（第151-152行）
3. 保存调试图像验证检测框位置

### Q3: 未读红点检测失败？

**原因：**
- 红点颜色与预设HSV范围不匹配
- 红点尺寸超出过滤范围

**解决方案：**
1. 使用取色器获取实际红点HSV值
2. 调整颜色范围（第17-20行）
3. 修改面积过滤条件（第80行）

### Q4: 导入错误 "No module named 'rapidocr_onnxruntime'"？

**解决方案：**
```bash
# 确认虚拟环境已激活
pip show rapidocr-onnxruntime

# 重新安装
pip install rapidocr-onnxruntime -i https://pypi.org/simple --no-cache-dir

# 检查Python版本（需3.8-3.11）
python --version
```

### Q5: 首次运行卡住不动？

**原因：** RapidOCR正在下载模型文件（约100MB）

**解决方案：**
- 耐心等待下载完成（取决于网络速度）
- 如下载失败，可手动下载模型文件并配置本地路径
